# Learn Git Branching을 통해서 git 배우기

## 개요

이번 단계에서는 지금까지 학습한 Git 개념을 기반으로  
원격 저장소(Remote Repository) 와의 협업 흐름을 학습했다.  
이전 단계까지는 로컬에서 브랜치와 커밋을 다루는 방법에 집중했다면,  

이번 단계에서는  

- 다른 사람이 작업한 내용을 가져오고  
- 내 작업을 공유하고  
- 충돌을 해결하는 과정  

즉, 실제 협업 상황에서의 Git 사용 방식이 핵심이었다.  
특히 단순 명령어 사용이 아니라  
“**언제 fetch / pull / push를 써야 하는지” 판단하는 능력**이 중요해지는 단계였다.

## Git Remote (원격 저장소)  

### 개념

원격 저장소는 네트워크 상에 존재하는 Git 저장소로  
여러 개발자가 함께 작업할 수 있도록 해주는 공간이다.  

### 포인트  

- 로컬 저장소와 별도로 존재  
- 협업 및 백업 역할  
- GitHub 등이 대표적인 예시

## Clone (원격 → 로컬)  

### 개념

- git clone  
  
원격 저장소를 로컬로 복사하는 명령어이다.  

### 특징

- 프로젝트 전체가 로컬에 생성된다.  
- 원격과 자동 연결된다.  
- o/main 같은 원격 브랜치 생성  

### 포인트

- clone = 시작점  
- 이미 협업 구조가 연결된 상태

## 원격 브랜치 (Remote Branch)

### 개념

- o/main
  
원격 저장소의 상태를 나타내는 브랜치  

### 특징
  
- 직접 수정 불가능  
- fetch/pull 시 자동 업데이트  

### 포인트

- 원격 상태를 “복사해서 보여주는 역할”  
- 로컬 main과는 별개
