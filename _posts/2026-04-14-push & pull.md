# Learn Git Branching을 통해서 git 배우기

## 개요

이번 단계에서는 지금까지 학습한 Git 개념을 기반으로  
원격 저장소(Remote Repository) 와의 협업 흐름을 학습했다.  
이전 단계까지는 로컬에서 브랜치와 커밋을 다루는 방법에 집중했다면,  

이번 단계에서는  

- 다른 사람이 작업한 내용을 가져오고  
- 내 작업을 공유하고  
- 충돌을 해결하는 과정  

즉, 실제 협업 상황에서의 Git 사용 방식이 핵심이었다.  
특히 단순 명령어 사용이 아니라  
“**언제 fetch / pull / push를 써야 하는지” 판단하는 능력**이 중요해지는 단계였다.

### 개념

원격 저장소는 네트워크 상에 존재하는 Git 저장소로  
여러 개발자가 함께 작업할 수 있도록 해주는 공간이다.  

### 포인트

## Clone (원격 → 로컬)  

### 개념

- git clone  
  
원격 저장소를 로컬로 복사하는 명령어이다.  

### 특징

- 프로젝트 전체가 로컬에 생성된다.  
- 원격과 자동 연결된다.  
- o/main 같은 원격 브랜치 생성  

### 포인트

## 원인

Git은 히스토리가 꼬이는 것을 방지하기 위해 **원격보다 뒤쳐진 상태에서는 push를 막는다.**  

---

## 해결 방법

### 방법 1

```python
git fetch
git rebase o/main
git push
```
  
내 커밋을 최신 위로 재정렬  

---

### 방법 2

```python
git fetch
git merge o/main
git push
```
  
merge commit 생성  

---

### 방법 3  

```python
git pull --rebase
git push
```
  
실무에서 가장 많이 사용
