# Learn Git Branching을 통해서 git 배우기

## 개요  

이번 단계에서는  
**원격 저장소를 다루는 고급 기능들**을 실제 문제 형태로 학습했다.  
  
이전 단계가 개념 위주였다면,

## Push Main

### 상황

- feature 브랜치에서 작업 완료
- main을 최신 상태로 유지해야 한다.

### 해결 흐름

- git pull --rebase
- git push
 
### 포인트

- pull 할 때 rebase 사용 → 히스토리 깔끔
- 최신 main 기준으로 작업 정렬 후 push

## 여러 브랜치 push (side1, side2, side3)  
  
### 상황  
  
- 여러 브랜치를 각각 push 해야 한다.
  
### 핵심
  
브랜치 하나씩 push 가능  
   
git push origin side1  
git push origin side2  
git push origin side3  
   
### 포인트  
  
- 원격 저장소는 각 브랜치를 따로 관리  
- 순서 중요 (히스토리 꼬일 수 있음)

---

## merge vs rebase (원격 작업)  
  
### 상황 

- 원격 변경사항을 반영해야 하는데 방식 선택 필요

---

### rebase 방식  

```python
git fetch  
git rebase o/main  
git push
``` 
 
**장점** - 히스토리가 깔끔하다. (선형 구조)  

---

### merge 방식

```python
git fetch  
git merge o/main  
git push
``` 
 
**특징**  
  
- merge commit 생성  
- 히스토리 복잡해진다.

## 차이  

rebase	  / 깔끔하다.
merge	  / 기록 보존

## git push 인자

### 기본 형태
   
- git push origin main
- local main → origin main

### 포인트  

#### 현재 위치 push  
   
git checkout C0  
git push origin main  
- HEAD 위치를 main으로 push

---

#### source : destination 

- git push origin foo:main  
- foo → origin main  

---

### 브랜치 생성
   
git push origin main:newBranch  
   
원격에 newBranch 생성  

---

### 요약  
  
main	  / 현재 브랜치 push  
foo:main	  / foo를 main에  
main:newBranch	   / 새 브랜치 생성

## 이번 단계에서 느낀 점

이번 단계는 단순 Git 명령어 학습이 아니라  
“**원격 저장소를 자유롭게 조작하는 단계**”였다.  
 
인상 깊었던 점:  

- push 실패 원리를 이해하게 된다.  
- rebase vs merge 차이를 체감했다.  
- source:destination 구조를 이해 했다.

## 정리  

Git은 단순 명령어가 아니라 “**그래프를 조작하는 도구**” 이다.
