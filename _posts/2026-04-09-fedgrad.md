# 2026 SPACE WAR - 01.ARIES CTF 문제 풀이

## 0. 개요

- **문제 이름** : fedgrad
- **문제 유형** : AI
- **환경** : 
  - 모델 : HuggingFaceTB/SmolLM2-135M  
  - 입력 : 단일 문자열 (hidden flag)  
  - 제공 파일:  
      - trainer.py  
      - proto.py  
      - gradients.pcap  
- **목표** :
  - **학습 과정에서 전송된 gradient를 분석하여 숨겨진 문자열 hspace{...}를 복원**

---

## 1. 전체 구조 분석

### 1-1. 서버 동작

trainer.py를 보면 학습 과정은 다음과 같다.  

문자열 (flag)  
→ tokenizer  
→ 모델 입력  
→ next-token prediction loss  
→ loss.backward()  
→ gradient 계산  
→ 모든 parameter gradient를 TCP로 전송  
   
즉, 모델이 한 문장을 학습하면서 생성된 gradient가 그대로 외부로 유출된다.  

---

### 1-2. 전송 포맷

proto.py 기준 gradient 직렬화 구조는 다음과 같다.  

```python
[name length]
[name]
[ndim]
[shape]
[data length]
[float32 raw bytes]
 ```
  
따라서 pcap을 분석하면 모든 파라미터의 gradient를 완전히 복원 가능하다.  

---

### 1-3. 문제 정의

이 문제는 다음과 같이 정리할수있다. **gradient만 보고 입력 문장을 복원할 수 있는가**

---

## 2. 모델 및 학습 구조

### 2-1. 모델  

사용 모델:  

HuggingFaceTB/SmolLM2-135M  
 
→ causal language model  

---

### 2-2. 학습 방식  

- 단일 문장 입력  
- next-token prediction  
- cross-entropy loss
  
즉, 일반적인 autoregressive language modeling  

---

### 2-3. 포인트 

이 구조에서 중요한 사실:  embedding gradient는 입력 토큰에만 강하게 나타난다  

---

## 3. 문제 핵심

### 3-1. 분석 대상

가장 중요한 텐서:  
model.embed_tokens.weight.grad  
  
이유:  
- 토큰별 embedding gradient 존재  
- 입력된 토큰 위치만 값이 크다.


  nonzero row = 사용된 토큰  

---

### 3-2. 아이디어

문제 해결 전략:  

1. pcap에서 gradient 복원  
2. embedding gradient 추출  
3. nonzero row 찾기  
4. tokenizer로 token → 문자열 변환

 입력 문장 일부 복원 가능  

 --- 

 ## 4. 공격 시나리오

 ### 4-1. gradient 복원  

 pcap → TCP stream 재조립 → tensor 파싱  

결과 :  embedding gradient matrix 복원  

---

### 4-2. 토큰 추출

nonzero row를 확인하면 다음 토큰들이 등장한다.  

```python
the
flag
is
h
space
{
_
f
s
hat
ter
data
priv
acy
ools
```

---

### 4-3. prefix 복원  

이 토큰들을 보면 이렇게 **the flag is hspace{** 완성된다.  

---

### 4-4. 내부 문자열복

토큰을 자연스럽게 합치면  

```python
fools
shatter
data
privacy
```

---

### 4-5. 최종 조합

```python
hspace{fools_shatter_data_privacy}
```

---

## 5. 중요한 특징 (BPE 토크 나이저)

### 5-1. 토큰 분리 문제  

토큰이 다음처럼 쪼개진다.  

```python
priv + acy
s + hat + ter
f + ools
```

즉, 문자열이 그대로 나오지 않는다.  

### 5-2. 해결 방법

- 토큰을 문자처럼 읽지 않는다.
- tokenizer 기준으로 재조합

자연어 문장으로 복원  

---

## 6. 전체 공격 흐름 정리

전체 과정은 다음과 같다.  

1. gradients.pcap 분석  
2. TCP stream 재조립  
3. gradient tensor 파싱  
4. embedding gradient 추출  
5. nonzero row 확인  
6. tokenizer로 token → 문자열 변환  
7. 문장 구조 추론  
8. flag 조합

--- 

## 7. 포인트

이 문제의 포인트는 다음과 같다.  

- gradient 자체가 정보 누출 채널  
- embedding gradient는 입력 토큰을 직접 드러낸다.  
- BPE 토큰을 올바르게 재조합해야 한다.  
- 모델을 깨는 것이 아니라 **학습 데이터를 복원하는 문제**

---

## 8. 정리

- language model의 embedding gradient를 분석하여 입력 문장을 역으로 복원하는 문제
